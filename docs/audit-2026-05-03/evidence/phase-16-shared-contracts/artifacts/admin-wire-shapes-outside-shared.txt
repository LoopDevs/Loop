Breadth indicator:
- rg found the same "promoting them to @loop/shared would just add indirection" pattern in multiple web service files: admin-activity, admin-merchant-drill, admin-user-fleet-activity, admin-stuck, admin-payment-method-activity, admin-cashback-config, admin-user-credits, admin-payouts-by-asset, admin-top-users, admin-merchant-stats, admin-payment-method-share, admin-monthly, admin-user-drill, admin-users-list, admin-user-cashback-by-merchant, and admin-merchant-activity.

Reasoning:
- These are not frontend-only types; they are response bodies produced by backend handlers and consumed by web service wrappers.
- Because backend and web compile against independent local interfaces, a backend field rename/type change can pass backend compilation and a web local type can remain stale until runtime/tests catch it.
- This contradicts the current shared-package rule and the surrounding pattern where admin treasury/assets/supplier/operator contracts were centralized in @loop/shared.
