    32	 * `Idempotency-Key`. The backend dedups against (user_id, key) so a
    33	 * double-clicked submit, a retried fetch, or a network-flap retransmit

    37	 * Generates a UUID v4 — 36 chars sits well inside the 16-128 server
    38	 * window, the entropy is far past anything a client double-click can

    40	 * supported browsers + the Capacitor webview.

    44	  // HTTPS in prod and on `capacitor://` in the native shell which
    45	  // counts as secure. Locally over `http://localhost` it's also OK.
    46	  return crypto.randomUUID();
