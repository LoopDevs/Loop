# Journey Maps

Journey maps force the audit to trace behavior across packages, files, storage, services, DB, workers, docs, tests, and operations.

Journey files:

- [user-journeys.md](./user-journeys.md)
- [admin-journeys.md](./admin-journeys.md)
- [data-money-journeys.md](./data-money-journeys.md)
- [operational-journeys.md](./operational-journeys.md)
- [adversarial-journeys.md](./adversarial-journeys.md)
- [planned-feature-journeys.md](./planned-feature-journeys.md)

Each journey must record:

- entry point
- actor and trust level
- frontend files
- backend files
- shared contracts
- DB tables and migrations
- external systems
- state transitions
- observability
- tests
- docs and runbooks
- failure modes
- findings or no-finding conclusion
