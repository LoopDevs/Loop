## P0 — Critical

**CF2-01 [LIVE] Operator circuit breaker can never self-heal.** `ctx/operator-pool.ts:226-237` filters `getState() !== 'open'` before `.fetch()`, but the OPEN→HALF_OPEN cooldown only runs _inside_ `.fetch()` (`circuit-breaker.ts:134-142`). One bad 401 strands an operator for the process lifetime; a second bad response bricks the 2-operator pool fleet-wide; `/health` bypasses breakers so Fly can't restart. Sources: v-ctx CTX-01, x-flows flow-1. Fix: `isAvailable()` filter + real-breaker integration test.

**CF-04 [LIVE·operational] Security-audit merge gate RED again.** `scripts/check-audit-policy.mjs` allowlist not kept current; `npm audit --omit=dev` already silences 2 of 3 dev-only highs; only `hono` (production framework) actually reaches runtime. Source: x-infra X-INFRA-01. Fix: allowlist dev-only highs + merge hono bump; make `--omit=dev` the primary signal; add a cron audit→oncall.

**CF-01 [API-LIVE, UI-blocked-only — not purely GATED] Redemption never burns inbound LOOP.** Because deposit==operator (confirmed, ADR 010 §"known limitation"), redeemed LOOP is the exact pool the payout worker draws for the _next_ user's cashback; drift watcher has no term for this and either inflates permanent drift or silently absorbs it. Burn fix exists only on `fix/adr036-emission-burn`. Sources: x-flows XFC-04, v-credits P1-01, v-wallet W-07. **Adversarially re-verified 2026-07-01 — severity corrected upward**: `loop_asset` is a fully-wired, non-flagged entry in `ORDER_PAYMENT_METHODS` and `loop-handler.ts` actively recommends it over the disabled `credit` method; the _only_ thing currently preventing exploitation is the web UI hardcoding `paymentMethod: 'usdc'` client-side, not a server-side flag. Any direct API caller with an on-chain LOOP-asset cashback balance can trigger this today in any deployment with cashback issuance active — this is not safely dormant behind `LOOP_WORKERS_ENABLED` alone. Tranche-2 blocker, but treat as urgent regardless of Tranche-1/2 framing given the API-reachability.

**CF-05 [BRANCH] Interest mint creates unbacked LOOPUSD/LOOPEUR + retired asset codes.** `interest-mint.ts` mints all three; ADR-031 mints on-chain only GBPLOOP. Source: v-wallet W-01/W-11. Do not merge Phase D. (v-wallet also rates W-02 — Privy auth-header missing → pipeline non-functional — a P0-BRANCH; folded into CF-32 cluster.)

**CF2-02 [LAUNCH-GATE] Zero terms/age-gate at signup.** Signup = email + OTP; no consent UI, no `termsAcceptedAt` column. Blocks App Store 5.1.1 + counsel sign-off. Source: x-privacy PRIV-05 (reopens CF-26).

**CF2-03 [LAUNCH-GATE] Zero sanctions/OFAC/geo screening** — signup, orders, or payout destinations; MaxMind is UX-only. ADR-035's AE/IN/SA order markets raise the stakes. Source: x-privacy PRIV-06 (reopens CF-26).
