## CRITICAL / HIGH (deduped)

### Accessibility (never audited — computed WCAG failures)

- **P2-01 `Button` focus-visible indicator fails WCAG 1.4.11 (~1.7:1 vs 3:1 required) — systemic.** `critical · LIVE` · `components/ui/Button.tsx:52`. The sole focus signal on every button variant is `ring-blue-500/40`; at 40% opacity over white it measures 1.74:1 (computed). Keyboard users can't reliably tell which button has focus — a real money-mistake vector on confirm/step-up dialogs. _Fix:_ drop the `/40` (solid clears 4:1) + add an axe test.
- **P2-02 Success-toast text fails WCAG 1.4.3 (3.22:1 vs 4.5:1) — every success confirmation.** `high · LIVE` · `components/ui/ToastContainer.tsx:4` (`bg-green-600 text-white`). The theme override pinned blue/gray but skipped green; `green-600`+white = 3.22:1. Same root cause makes `Badge` solid warning/success fail (2.13 / 3.22). _Fix:_ `green-700`/`amber-700`, or extend the `@theme` pin.

### Client-side security

- **P2-03 `redeemUrl` rendered as a raw `<a href>` with no scheme validation → `javascript:` XSS in the native webview.** `high · GATED` · `components/features/purchase/LoopPaymentStep.tsx:230`, `orders/LoopOrdersList.tsx:141`. The _same_ value is routed through `assertSafeUrl` on the `RedemptionBlock`/`openWebView` path but bound raw here (AGT-11 seam). `z.string().url()` accepts `javascript:`, so backend validation isn't a block; the native meta CSP keeps `unsafe-inline`, so a hostile-CTX `javascript:` redeemUrl executes on tap. _Fix:_ scheme-gate the anchor or route all three consumers through `openWebView`.
- **P2-04 WebView script injection is origin-scoped on the inbound message but NOT the outbound injection → redeem code leaks cross-origin.** `high(med-adj) · LIVE` · `native/webview.ts:106-112` vs the origin check at `:84`. The challenge-bar (which writes the redeem code into the page DOM) is re-injected on _every_ `browserPageLoaded` for _every_ origin; a cross-origin redirect during redeem (merchant→SSO) plants the code into a foreign origin's DOM. _Fix:_ gate `executeScript` on `currentOrigin === allowedMessageOrigin`.
- **P2-05 Backend OTP consumption is non-atomic → one "one-time" code can mint two token pairs.** `med→high · LIVE · BACKEND · CROSS-PHASE double-find` · `apps/backend/src/auth/otps.ts:87-122`. Plain `SELECT` + unconditional `UPDATE ... SET consumed_at` with **no `WHERE consumed_at IS NULL` CAS and no rowcount check** — unlike the codebase's own refresh-token rotation which does. Two concurrent verifies both pass `findLiveOtp` before either write lands. **Phase 1 already found this** (its concurrency long-tail, at the caller `auth/native.ts:104-133`, rated medium); this Phase-2 finder independently re-found it one layer down at the DB and rated it high after noticing a _client-side_ double-verify race on native onboarding (`Onboarding.tsx:243`) that makes it accidentally triggerable, not just adversarially. Two phases hitting the same bug from opposite ends is strong corroboration; I reconcile the severity at **high-leaning-medium** (the code is still user-held, so it's a one-time-guarantee violation + accidental double-session, not an outsider auth bypass). _Fix:_ `UPDATE ... WHERE consumed_at IS NULL RETURNING id` + rowcount check server-side; synchronous `useRef` submit-guard client-side.

### Admin UI

- **P2-06 Shared single-slot step-up queue drops one of two concurrent money-write confirmations (confused-deputy).** `high · LIVE` · `hooks/use-admin-step-up.ts:56-113`, consumed once-per-page by list routes (`admin.payouts.tsx:134`, `admin.skips.tsx:108`, `admin.staff.tsx:58`). A second gated action while one is queued **overwrites** the pending slot unconditionally; the first's promise is orphaned (never resolves), and the operator's OTP authorizes the _other_ action — with the modal showing no context about _what_ it authorizes (**P2-07**, `StepUpModal.tsx:114` — an irreversible emission's amount/destination isn't echoed). _Fix:_ key the pending slot per action-id (reject the superseded one loudly); scope the hook per-action; echo the payload in the modal.

### i18n / SEO (never audited)

- **P2-08 i18n framework landed but the high-traffic chrome was never wired to it.** `high · LIVE` · Navbar, CountrySelector, `trustlines.tsx`, `CashbackCalculator` have **zero** `useTranslation` — every string is a literal. `SUPPORTED_LANGS=['en']`; adding a language needs a code rewrite, not a JSON drop (contradicting the design goal). _Fix:_ route chrome through `t()` (Footer is the template).
- **P2-09 `CashbackCalculator` hardcodes a `$` glyph regardless of the merchant's real currency.** `high · LIVE · double-find` (i18n + client-flows) · `CashbackCalculator.tsx:98`. Input shows `$`, output shows `£`/`€`/`د.إ` — the widget contradicts itself, though the correct `data.currency` is in scope. _Fix:_ `currencySymbol(data.currency, locale)`.
- **P2-10 The purchase-conversion pages (`/gift-card/:name`, `/brand/:slug`, `/map`) have no sitemap entry AND no `<link rel=canonical>`.** `high · LIVE` · `sitemap.tsx:105`, `map/gift-card/brand meta()`. Reachable at a legacy URL + up to 28 locale-prefixed URLs with zero canonical → duplicate-content on the most commercially important page type. Every sibling route calls `canonicalHref`. _Fix:_ add canonical + `localizedPage()` treatment.
- **P2-11 `/cashback/:slug` (and `/gift-card/:name`, `/brand/:slug`) is a soft-404.** `high · LIVE · double-find` (i18n SEO-04 + client-flows AUD-18) · no `loader`; `meta()` fabricates a plausible title/description from the raw slug; HTTP always 200. Doorway-page abuse + crawl-budget dilution. The `not-found-ssr` fix (`throw 404`) wasn't applied to the dynamic landing pages. _Fix:_ server-side existence check in a loader → true 404.

### Config / CI / tests

- **P2-12 The `entry.server.tsx` security-header wiring is untested and excluded from coverage.** `high · LIVE` · `vitest.config.ts:39` excludes the file; `entry-server-headers.test.ts` re-tests a helper, not the handler; no e2e asserts a live response header. The A-027 regression (prod once shipped with zero security headers) could reland green. _Fix:_ one `page.on('response')` header assertion in the live smoke test; remove the coverage exclusion.
- **P2-13 The e2e suites covering the REAL purchase path and the admin dashboard are not required-to-merge.** `high · LIVE` (confirmed via live `gh api`) · only `test-e2e-mocked` (legacy CTX proxy) is in `required_status_checks`; `test-e2e-flywheel` (loop-native purchase + admin) is not. A regression in `createLoopOrder` or the admin step-up gate can merge behind an ignorable red X. _Fix:_ add `E2E tests (loop-native flywheel)` to required checks.

### Frontend structural absence

- **P2-14 No min-app-version / forced-update gate.** `high · NEEDS-CODE · double-find` (mobile MBL-07 + negspace A1) · the client stamps `X-Client-Version`, the backend only logs it; a shipped/sideloaded Capacitor bundle can't be OTA-corrected and can't be refused. Compounds the ADR-027 mobile absences. _Fix:_ `minSupportedVersion` in `/api/config` + a `ForceUpdateGate` in the native branch.

## MEDIUM (grouped)

- **Mobile/native:** "prevent screenshots" overclaim (no `FLAG_SECURE`, 30%-opaque JS overlay, Android recents captured pre-`pause`); app-lock is off-by-default, fail-open, cold-start-only DOM theater over already-rendered content; OS App-Link/Universal-Link verification 404s by default (a malicious app can intercept links incl. the Apple-Sign-In callback); no TLS pinning/attestation (ADR-027 deferred) with compensating controls materially weaker than the ADR claims; clipboard writes gift-card code/PIN with no auto-clear.
- **Client-side security:** step-up 401 misread as access-token expiry → a transient refresh failure wipes the admin session mid-write (`api-client.ts:266`); the `isFresh()` step-up freshness guard is **dead code** (double-find — never read, so every stale token pays a wasted round-trip); CSP `<meta>` never carries the per-request nonce so SSR advertises `unsafe-inline` (masked by header intersection, double-find F4/M5); refresh token + email in web `sessionStorage` (script-reachable; native uses Keychain).
- **Admin UX/a11y:** web admin flashes "Sign in" to authed staff on hard reload (restore-splash is native-only); `±10M` money-write cap duplicated 5× with no shared constant (double-find with recon — a CI-catchable drift); `retryingId` single scalar desyncs per-row loading (double-submit path); `AdminAuditTail` renders 5-column data as unlabeled flex rows; `ReasonDialog`/`StepUpModal` errors not in a live region + focus doesn't return; `StepUpModal` OTP field missing `autocomplete="one-time-code"`; `failureReason`/raw Horizon payment JSON rendered to the broader _support_ tier (PLAUSIBLE — verify backend sanitization).
- **i18n/format:** `formatCashbackMinor`/`MerchantCard` never pass locale → always en-US formatting; denomination ranges bypass `Intl.NumberFormat` (no thousands separators — wrong even in en); zero RTL CSS despite AED/SAR being routed markets (physical `pl-`/`pr-`, no logical `ps-`/`pe-`); SSR `<html lang>` hardcoded `en` with a comment overclaiming the gap is closed (AGT-08); the `formatMinorCurrency` 2-decimal split breaks on a future 3-decimal currency (KWD/BHD — double-find AUD-13).
- **SEO/UX:** meta promises content that `Phase2Gate` hides today (crawler indexes an oversell — add `noindex` while gated); `Phase2Gate` "back to directory" drops the visitor's locale; `/styleguide` reachable in prod with no gate (only `noindex`, not in `robots.txt` Disallow); home/brand grid renders ~1,134 records un-virtualized; `AUD-07` conversion CTA links `/gift-card/:id` instead of `:slug` (dead link on the funnel); `AUD-08` client pending-order filter runs after server pagination → false "no orders" empty state; `AUD-10` wallet/pay-with-loop silently vanish on a transient fetch error with no retry.
- **UI primitives:** Toast dismiss button below the 24×24 WCAG 2.5.8 target size; `Skeleton` loaders silent to screen readers (unlike `Spinner`); `Card interactive` prop implies clickability with no keyboard/role scaffolding (latent `div onClick`); `redeem-challenge-bar` Copy button state not announced; no shared `Dialog` primitive (three hand-rolled `<dialog>` wrappers → the a11y fix must be applied N times).
- **Config/tests:** the pinned `vite@8.0.16` is not the version tests run on (root vitest resolves `vite@7.3.2`) → Vite-8 behavior untested; the one real-purchase-path e2e is a single happy-path test (no wrong-amount/expiry/USDC); `StepUpModal.test` covers no error branches; no CI-driven deploy for the web app (deployed artifact ≠ CI-verified build); web Fly health check probes `/` (a geo-redirect that round-trips to the backend + can 302 → couples web liveness to backend health); onboarding `es`/`pt` locale defaults to EUR (wrong for LatAm/Brazil, latent).
- **Frontend absences:** no centralized session-expiry UX on web (a mid-purchase 401 shows a generic toast, not "sign in again"); no route error-boundary on the wallet/settings money screens (a render crash blanks the whole native app with no tab bar); no `Retry-After` countdown on 429 (user re-taps into the limit); money-action buttons stay enabled on network loss.

## LOW / INFO (categories)

- **Correctness/dead-code:** duplicated formatters (`formatCashbackPct` byte-identical in two files, `settings.cashback#formatAmount` re-implements the canonical bigint formatter); `hasSufficientCredit`-style comment/code drift; pervasive dead `dark:` classes (dark mode retired); `Avatar` `alt=""` on empty-string name (`??` vs `||`); `Button` allows an icon-only instance with no accessible-name guard.
- **Onboarding a11y (native-only per the scope correction — the web `/onboarding` uses `OnboardingDesktop`):** OTP 6-box grid inputs unnamed, error text no `role="alert"`, "resend code" no cooldown feedback, animations ignore `prefers-reduced-motion`, progress dots visual-only, biometric-failure silent; the `inert` a11y fix silently defeated the Android soft-keyboard focus call (AGT-11); a hand-rolled email regex duplicated in two files, weaker than the backend's.
- **Copy/consistency:** `INSUFFICIENT_BALANCE` copy is developer-voiced; the `/auth` vs `/onboarding` error-copy divergence a comment claims was unified (`authErrorMessage` never exported/wired, AGT-08); 429 copy conflates per-IP throttle with per-email 15-min lockout and drops `Retry-After`; bundle-budget CI comment ~70% stale; stale `@capacitor/push-notifications` artifacts in the generated native tree (not a dependency).
- **Info:** no runtime opt-out for crash/perf telemetry (cookieless posture keeps it PECR-exempt; GDPR legitimate-interest to watch).
