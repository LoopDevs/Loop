# @loop/mobile

Capacitor v8 shell for iOS and Android. Loads the static web build from `apps/web/build/client/`.

## Prerequisites

- iOS: Xcode 15+, iOS 16+ target
- Android: Android Studio, API 35+ target
- Built web assets: `cd apps/web && npm run build:mobile`

## Workflow

```bash
# 1. Build the web static export
cd apps/web && npm run build:mobile

# 2. Sync to native projects
cd apps/mobile && npx cap sync

# 3. Re-apply audit overlays (A-033 backup rules, A-034 Face ID usage
#    description). Idempotent; run after every `cap sync` so audited
#    config survives the template regeneration — see ADR-007.
./scripts/apply-native-overlays.sh

# 4. Open native IDE
npx cap open ios        # Xcode
npx cap open android    # Android Studio
```

## Live reload (dev)

1. Temporarily add to `capacitor.config.ts`:
   ```typescript
   server: { url: 'http://<your-local-ip>:5173', cleartext: true }
   ```
2. `cd apps/web && npm run dev`
3. `cd apps/mobile && npx cap sync && ./scripts/apply-native-overlays.sh && npx cap open ios`
4. Remove `server` block before committing.

## Native projects

- `ios/` — Xcode project (gitignored, regenerated by `cap add` / `cap sync`)
- `android/` — Android Studio project (gitignored, regenerated by `cap add` / `cap sync`)

Config comes from two sources, both versioned:

- `capacitor.config.ts` — bundle id, plugin config, server-url dev overrides
- `native-overlays/` + `scripts/apply-native-overlays.sh` — Android backup
  exclusions (audit A-033) and iOS `NSFaceIDUsageDescription`
  (audit A-034). See ADR-007 for the rationale and the bootstrap flow.

**Never hand-edit files under `ios/` or `android/`** — they're regenerated.
Changes belong in one of the two versioned sources above.
